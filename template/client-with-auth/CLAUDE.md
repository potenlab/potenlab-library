# Client With Auth Template — CLAUDE.md

## What is this?

A Next.js starter template with shadcn/ui authentication pages (login/register) and a minimal sidebar dashboard. All UI components come directly from shadcn/ui — this template does NOT use `@potenlab/ui`.

## Stack

Next.js (App Router) | TypeScript | Tailwind CSS v4 | shadcn/ui (New York) | Lucide Icons

## Key Patterns

1. **Auth pages are standalone** — `/login` and `/register` render outside the dashboard layout (no sidebar).
2. **Dashboard uses a layout** — `src/app/dashboard/layout.tsx` wraps all `/dashboard/*` routes with the sidebar shell.
3. **Feature-based architecture** — Business logic lives in `src/features/`, app routes are thin wrappers that import from features.
4. **All UI is shadcn/ui** — Components in `src/components/ui/` are generated by shadcn CLI. Don't manually edit them.
4. **No auth logic yet** — This is a UI template. Auth backend (NextAuth, Clerk, etc.) must be wired in by the developer.

## Routes

| Route | Purpose |
|---|---|
| `/` | Redirects to `/login` |
| `/login` | Login page |
| `/register` | Register page |
| `/dashboard` | Dashboard home |
| `/dashboard/analytics` | Analytics page |
| `/dashboard/users` | Users page |
| `/dashboard/content` | Content page |
| `/dashboard/settings` | Settings page |

## Commands

```bash
npm install          # Install dependencies
npm run dev          # Start dev server
npm run build        # Production build
npm run lint         # Run ESLint
```

## Adding shadcn Components

```bash
npx shadcn@latest add <component-name>
```

Components are installed into `src/components/ui/`.
