# Client With Auth Template — CLAUDE.md

## What is this?

A Next.js starter template with shadcn/ui authentication pages (login/register) and a minimal sidebar dashboard. All UI components come directly from shadcn/ui — this template does NOT use `@potenlab/ui`.

## Stack

Next.js (App Router) | TypeScript | Tailwind CSS v4 | shadcn/ui (New York) | Lucide Icons

## Project Structure

```
app/
├── layout.tsx                    → Root layout (Geist fonts, TooltipProvider)
├── page.tsx                      → Redirects to /login
├── globals.css                   → Tailwind v4 + shadcn CSS variables
├── login/page.tsx                → Login page
├── register/page.tsx             → Register page
└── dashboard/
    ├── layout.tsx                → Dashboard shell (SidebarProvider + AppSidebar)
    ├── page.tsx                  → Dashboard home (stat cards)
    ├── analytics/page.tsx        → Analytics placeholder
    ├── users/page.tsx            → Users placeholder
    ├── content/page.tsx          → Content placeholder
    └── settings/page.tsx         → Settings placeholder

components/
├── login-form.tsx                → Login card form
├── register-form.tsx             → Register card form
├── app-sidebar.tsx               → Sidebar navigation with icons
└── ui/                           → shadcn/ui primitives (DO NOT edit directly)

hooks/
└── use-mobile.ts                 → Mobile detection hook (from shadcn)

lib/
└── utils.ts                      → cn() utility
```

## Key Patterns

1. **Auth pages are standalone** — `/login` and `/register` render outside the dashboard layout (no sidebar).
2. **Dashboard uses a layout** — `app/dashboard/layout.tsx` wraps all `/dashboard/*` routes with the sidebar shell.
3. **All UI is shadcn/ui** — Components in `components/ui/` are generated by shadcn CLI. Don't manually edit them.
4. **No auth logic yet** — This is a UI template. Auth backend (NextAuth, Clerk, etc.) must be wired in by the developer.

## Routes

| Route | Purpose |
|---|---|
| `/` | Redirects to `/login` |
| `/login` | Login page |
| `/register` | Register page |
| `/dashboard` | Dashboard home |
| `/dashboard/analytics` | Analytics page |
| `/dashboard/users` | Users page |
| `/dashboard/content` | Content page |
| `/dashboard/settings` | Settings page |

## Commands

```bash
npm install          # Install dependencies
npm run dev          # Start dev server
npm run build        # Production build
npm run lint         # Run ESLint
```

## Adding shadcn Components

```bash
npx shadcn@latest add <component-name>
```

Components are installed into `components/ui/`.
